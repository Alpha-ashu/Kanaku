BUG #1: Missing Sanitization in Account Update
────────────────────────────────────────────────
Location: backend/src/modules/accounts/account.controller.ts
Severity: HIGH (XSS Prevention)
Status: ✅ FIXED

Problem: String fields (name, provider, country) not sanitized during update
Impact: XSS payloads could be stored and executed
Fix: Added sanitization for all string fields in updateAccount()

Before:
  for (const field of allowedFields) {
    if (body[field] !== undefined) updates[field] = body[field];
  }

After:
  for (const field of allowedFields) {
    if ((field === 'name' || field === 'provider' || field === 'country')
        && typeof body[field] === 'string') {
      updates[field] = sanitize(body[field]);
    } else {
      updates[field] = body[field];
    }
  }

                        VALIDATION PATTERNS
================================================================================

All numeric validations follow this pattern:

  // For positive amounts/values
  if (body.field !== undefined) {
    const numValue = Number(body.field);
    if (!Number.isFinite(numValue) || numValue <= 0) {
      return res.status(400).json({ error: 'Field must be a positive number' });
    }
  }

  // For non-negative amounts (balances)
  if (body.field !== undefined) {
    const numValue = Number(body.field);
    if (!Number.isFinite(numValue) || numValue < 0) {
      return res.status(400).json({ error: 'Field must be non-negative' });
    }
  }

All string field sanitization uses:

  if ((field === 'name' || field === 'otherStringField') && typeof body[field] === 'string') {
    updates[field] = sanitize(body[field]);
  } else {
    updates[field] = body[field];
  }
