# KANKU API Testing Guide

Use the live docs generated by the backend:

- Swagger UI: `/api-docs`
- OpenAPI JSON: `/api-docs/openapi.json`
- Markdown guide: `/api-docs/testing-guide`

## Recommended QA flow

1. Authenticate with `POST /api/v1/auth/login` or `POST /api/v1/auth/register`
2. Save the bearer token
3. Test authenticated modules with:
   - `Authorization: Bearer <token>`
   - `Content-Type: application/json`
4. For every write flow, verify both:
   - API response correctness
   - persisted state in database / follow-up GET endpoints

## Feature groups to cover

- Auth
- Accounts
- Transactions
- Goals
- Loans
- Investments
- Friends
- Groups
- Settings
- Notifications
- Sync
- Dashboard
- Advisors / Bookings / Sessions
- Admin

## Method coverage

- `GET`: list, detail, filtered reads, unauthorized access
- `POST`: valid create, invalid payload, duplicate/business-rule cases
- `PUT` / `PATCH`: full update, partial update, invalid id, invalid state transition
- `DELETE`: delete success, deleted resource absent on follow-up reads, repeated delete behavior

## Critical integration checks

- Login -> profile -> dashboard bootstrap
- Add account -> account list refresh
- Add transaction -> account balance / dashboard updates
- Loan payment -> outstanding balance update
- Booking accept -> advisor session creation
- Session message -> persisted chat message + notification side effect
- Sync pull / push -> changed records reflected correctly
