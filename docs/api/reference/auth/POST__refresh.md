## Request

Rotates the access + refresh token pair. The credential is the **refresh token**,
supplied platform-specifically (see Project Overview §E.1):
- **Web:** the `HttpOnly` cookie `kanaku_rt`, sent automatically with
  `credentials: 'include'`. No body/header needed.
- **Native:** the stored token in the `x-refresh-token` request header, plus
  `X-Client-Platform: native`.

Body is empty (`{}`).

## Responses

Standard envelope: `{ "success": true, "data": … }` on success, `{ "success": false, "error": "…", "code": "…" }` on error.

On `200`, `data` contains `accessToken` and `expiresAt`; the new access token is
also in the `Authorization` header. The rotated **refresh token** is set as the
`kanaku_rt` HttpOnly cookie (web) and additionally returned as `data.refreshToken`
for native clients only. The legacy `x-refresh-token` response header is no longer sent.
