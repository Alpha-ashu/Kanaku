# Platform — System-Level Services

> The **platform layer** = the shared, cross-feature services every feature stands
> on: identity/security, the database, Supabase, configuration, and shared code.
> This folder is the **navigation hub**. Some real implementations stay at their
> current paths because deploy tooling, CLIs, or codegen require them there — each
> is linked below with the reason.

## What lives where

| Platform concern | Index here | Real implementation | Why it stays put |
|---|---|---|---|
| **Security** (auth, RBAC, validation, rate limit, audit) | [`security/`](./security/README.md) | `backend/src/middleware/`, `backend/src/security/`, `backend/src/features/auth/` | Compiles inside backend `rootDir: ./src`. |
| **Shared code** (utils, types, logger, infra) | [`shared/`](./shared/README.md) | `backend/src/utils/`, `frontend/src/lib/`, `frontend/src/utils/` | Inside each app's build graph. |
| **Database** (raw SQL, bootstrap) | _link →_ [`../database/`](../database/README.md) | `database/`, `backend/prisma/schema.prisma` | `backend/apply_schema.cjs` reads `../database/*.sql`; `database/docs/` is generated by `scripts/gen-catalogs.mjs`. |
| **Supabase** (managed Postgres/Auth/Storage) | _link →_ [`../supabase/`](../supabase/README.md) | `supabase/` | **Supabase CLI requires `supabase/` at repo root** (config.toml, migrations, edge functions). |
| **Config / secrets resolver** | _link →_ [`../config/`](../config/credentials.ts) | `config/credentials.ts` | Imported by app code via relative paths. |

> **Why `database/` and `supabase/` are both kept (they are NOT duplicates):**
> `supabase/` is the **Supabase CLI's own project directory** (hosted Postgres/Auth/Storage + edge functions + migrations the CLI manages). `database/` holds **self-hosted raw SQL** (`init.sql`, `supabase_schema.sql`, helper scripts) read directly by `backend/apply_schema.cjs`. They serve different deployment targets; merging them breaks both the CLI and the bootstrap script.

## Non-negotiables (from the security contract)

- API routes versioned under **`/api/v1`**.
- All protected routes require auth middleware; all mutating routes require zod validation.
- Monetary logic is **server-authoritative**; coupled balance updates use Prisma `$transaction`.
- Ownership check before every user-scoped read/write.
- Helmet + CORS + rate limiting never bypassed. No secrets in code. No `any` in new code.

See [`security/README.md`](./security/README.md) for the full controls catalog.
